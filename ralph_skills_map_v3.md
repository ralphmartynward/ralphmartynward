## BUILD BRIEF (handoff to development)

### Architecture
- **Subdomain**: `story.ralph-ward.com` (or `v2.`) — keep main site live during build, redirect on launch
- **Stack**: React + TypeScript + Vite, Tailwind for styling, GSAP + ScrollTrigger for scroll animations, D3 / react-force-graph for the skills network
- **Hosting**: static deployment (Vercel or Netlify), no backend needed for v1
- **Performance**: respect `prefers-reduced-motion`, lazy-load below-fold chapters, optimise graph rendering

### Content layer (locked in this document)
- 6 chapters of prose (Ch0 through Ch5), v3 in `ralph_chapter_prose_v3.md` *(send across when ready for build kick-off)*
- 40 skill nodes
- 7 capability clusters
- ~150 tools mapped to skills
- Edge annotations for project-level callouts in Chapter 4
- Tooltip option C: tools grouped by role/period and side project

### Build sequence
1. **Static MVP** — chapters as plain scroll, no animation. Proves words work on the page.
2. **Skills graph as standalone page** — easier to debug than embedding in scroll first. Get the layout, clustering, hover/tap behaviour right.
3. **Integrate graph into scrollytelling** — chapter scroll triggers graph state changes. Cross-cutting nodes scale up gradually (no pulse for v1).
4. **Mobile fallback** — simplified graph snapping cleanly between chapter states, not animating per-scroll. Tooltip becomes bottom-sheet.
5. **Polish, accessibility, performance**
6. **SEO & meta** — schema.org Person markup, OG cards per chapter, sitemap

- Growth: warm yellow / ochre
- Product: teal
- Ops: slate blue
- Data & AI: deep blue
- Eng: purple
- Biz: terracotta

7. Analytics on the new site — Plausible? GA? None? My recommendation: Plausible, lightweight, privacy-respecting, fits the values angle.

When the seven items above are answered, the build can start.
