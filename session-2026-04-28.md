### 2. Split-screen scrollytelling integration (Step 3)

Restructured the homepage from full-width chapters to a two-column layout where the graph tells the story alongside the prose.

**New files**
- `src/data/chapterActivation.ts` — per-chapter cumulative active node sets (Ch0–Ch5), cross-cutting node IDs, per-chapter scale multipliers for always-visible nodes
- `src/hooks/useActiveChapter.ts` — IntersectionObserver hook returning active chapter index; returns -1 at hero, 0–5 for chapters; uses scroll listener to reset to -1 when scrolling back to top

**`src/components/ChapterNav.tsx`** — refactored: active chapter state lifted to parent via prop (was computed internally). Removed fixed positioning; now rendered inside the graph panel.

**`src/pages/HomePage.tsx`** — full rewrite to split-screen layout:
- Left panel: `position: sticky; top: 0; height: 100vh; width: 58%` — contains graph, chapter nav dots, node tooltip, "explore all skills" link
- Right panel: `flex: 1` — Hero + ChapterSection map + footer; window-level scroll so existing IntersectionObserver still works
- Node hover tooltip: bottom-left of graph panel, shows cluster name, skill name, first 8 tools

**`src/components/ChapterSection.tsx`**
- Removed `lg:pl-24` and `lg:ml-16 xl:ml-32` (no longer needed without the fixed left nav)
- Reduced faded chapter number from `text-[14rem]` to `text-[8rem]` for the narrower column
- Simplified inner div to `max-w-xl`

**`src/components/SkillsGraph.tsx`** — major update: chapter-aware animated rendering
- New optional `chapter?: number` prop; -1=hero, 0-5=story mode, undefined=explore mode (no change to `/skills` page)
- Per-node animation state: `{ scale, opacity, labelAlpha }` lerped in RAF loop at 0.08/frame
- Per-edge animation state: `drawProgress` 0→1, draw-on via `setLineDash([len*progress, len+1])`
- Node tiers: active=scale 1.0/opacity 1.0, cross-cutting-not-yet-active=scale grows per chapter/opacity 0.65, inactive=scale 0.25/opacity 0.2/no label
- `chapterRef.current = chapter` in render body so RAF closure always reads current chapter without stale closure
- Separate `applyChapterTargets(ch)` function called on chapter change detection in RAF loop
- Canvas sizes to container (`canvas.offsetWidth/Height`) so it works in the 58% sticky panel and in full-screen explore mode
- Chapter-aware zoom-to-fit: tighter padding in story mode, legend-panel padding in explore mode

### 3. Hero pre-story state (chapter -1)

At the hero section (before any chapter enters the viewport), the graph shows all 40+ nodes as small faded dots (scale 0.3×, opacity 0.15, no labels, no edges). As the user scrolls into Ch0, nodes animate up.

- `useActiveChapter` initialises to -1; scroll listener resets to -1 when Ch0 top is still below 50% of viewport
- `computeNodeTarget(nodeId, -1)` returns `{ scale: 0.3, opacity: 0.15, labelAlpha: 0 }` for all nodes
- `isEdgeActive(link, -1)` returns `false` — no edges drawn at hero

### 4. Impact node convergence web (Ch5)

In Ch5, a special luminous node (node 41 "Impact") appears at the bottom-centre of the graph. Faint white lines draw from every visible node to it, making the "generalism made purposeful" idea literal.

- Node 41 positioned at `CLUSTER_POS.impact = { x: 0, y: 290 }` with high anchor strength (0.85) so it holds position without simulation edges
- No entries in `skillLinks` — convergence lines are drawn as a special canvas pass between the link loop and node loop
- Line opacity scales as `impactOpacity × 0.09 × nodeOpacity` per connecting node — builds up gradually as Ch5 activates
- Node 41 rendered with `shadowBlur: 20` at rest / `32` on hover — the most luminous node in the graph

**Ch5 accent colour** changed from `#e07a5f` (same as Business/biz cluster — confusing) to `#4ade80` (mint green — distinct from all 7 existing cluster colours).
